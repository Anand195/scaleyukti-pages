Keep your existing AiSensy setup. Configure a new bot flow for the subscription product. Webhook receives user message → trigger n8n. Set up template messages for content delivery (utility category = cheaper at ₹0.115/msg vs ₹0.863/msg for marketing).

Data Layer — PostgreSQL

Your saamarthdb or a new DB. New tables needed: users, subscriptions, credit_ledger, content_log, brand_kit. Subscription status checked on every request. Content logs maintained for audit (DPDP compliance) and retry logic.

EXTEND EXISTING
🧠

AI Layer — Claude API

Use Sonnet 4.6 for Starter/Growth plans (quality matters for positioning). Use Haiku 4.5 for Agency plan's high-volume requests (200+ packs/month). Enable prompt caching for system prompt (90% cost reduction on cached tokens). Batch API for non-real-time requests.

KEY COST LEVER
☁️

Cloud — GCP or AWS Mumbai

Your current pipeline runs on Cloud. Do NOT move it. Add a lightweight API gateway in front of the Social Plugin for production traffic management. GCP is preferred (10-20% cheaper, sustained use discounts, AI/ML native). Start with a single t2.micro equivalent + Redis cache + managed PostgreSQL.

RECOMMENDED: GCP
💳

Subscription Management

Two options: (1) WhatsApp Pay native (best UX, India only). (2) Razorpay subscription API (more reliable, webhook-driven, credit assignment on payment confirmation). Recommend Razorpay for initial launch due to reliability and webhook clarity. Add WhatsApp Pay as secondary later.

DPDP + GDPR Compliance Roadmap

India's DPDP Rules notified November 13, 2025 — 18-month phased rollout underway

Critical — DPDP Act Status (April 2026) DPDP Rules were notified November 13, 2025. Phase I (Data Protection Board setup) is effective immediately. Phase II obligations for businesses are rolling out. Breach reporting (72-hour window), consent requirements, and security safeguards are NOW active. Non-compliance can result in penalties up to ₹250 Crore.
DPDP Act 2023

India Data Protection

  • Collect explicit consent before processing any personal data (name, WhatsApp number, brand info)
  • Display privacy notice in plain language — what data, why, how long
  • Implement 72-hour breach notification to Data Protection Board + affected users
  • Encrypt all personal data at rest and in transit (AES-256 + TLS 1.3)
  • Maintain access logs and audit trail (required for significant data fiduciaries)
  • Honor data deletion requests within 30 days
  • Do NOT store data outside India if subject to localization requirements
  • Define data retention policy — max 90 days for content logs unless user opts in
GDPR

Global (If EU Users)

  • Add "Lawful Basis" documentation — consent is primary basis under both GDPR and DPDP
  • Implement right to access, rectification, erasure (same as DPDP but stricter)
  • Data Processing Agreement with all sub-processors (Anthropic, AiSensy, Razorpay)
  • Cookie consent if you have a web dashboard
  • Cross-border transfer mechanisms (Standard Contractual Clauses)
  • Privacy by Design — minimal data collection from day one
ISO 27001

Phase 2 Goal (Year 2)

  • Information Security Management System (ISMS) framework
  • Risk assessment and treatment plan
  • Access control policies for all systems
  • Incident response plan and playbook
  • Annual internal audit + external certification audit
  • Signals enterprise readiness — critical for B2B enterprise sales
SOC 2 Type II

Phase 3 Goal (Year 3)

  • Required for enterprise contracts, especially US/EU clients
  • Covers Security, Availability, Confidentiality trust principles
  • 12-month observation period before audit
  • Estimated cost: $15,000–$50,000 for first audit
  • Required for Series A enterprise positioning

Critical Questions & Recommendations

Every important question answered honestly, plus founder-level suggestions

Should I use token-based or credit-based subscription model?

Credit-based (Content Packs) is clearly superior for your use case.

Token-based pricing requires users to understand what a token is — confusing for non-technical Indian SME owners. Credit-based is intuitive: "You have 20 content packs this month." Everyone understands that.

Operationally, credits also give you cost predictability — you know exactly how much Claude API you'll consume per user tier. Token-based billing requires dynamic cost tracking which is complex to build.

Recommendation: 1 credit = 1 complete content pack (IG + LinkedIn + Twitter + Blog + Research). Users buy 20, 60, or 200 credits per month. Credits expire in 45 days (not the end of the month) — gives users flexibility without month-end usage spikes.

WhatsApp Pay vs Razorpay — which to use for subscriptions?

Razorpay Subscriptions for launch. WhatsApp Pay as additional option later.

WhatsApp Pay's in-app purchase features for SaaS subscriptions are still limited in India as of April 2026. WhatsApp Pay works for one-time payments but subscription management (auto-renewal, plan upgrades, webhooks for credit assignment) is not native.

Razorpay Subscriptions: (1) Full webhook support — credit assignment is instant on payment confirmation, (2) Handles retry logic for failed payments, (3) Plan management is clean, (4) UPI + cards + net banking all supported, (5) 2% transaction fee is standard.

WhatsApp Pay as a payment option inside Razorpay's checkout — yes. WhatsApp Pay as the subscription engine — not yet ready.

How do I handle the visual/carousel generation component at scale?

Use HTML-to-image conversion (Puppeteer/wkhtmltopdf) rather than image generation APIs.

Your Social Plugin already generates HTML. Converting HTML carousels to PNG images using a headless browser (Puppeteer on your VPS) is: (1) Extremely cheap (effectively free), (2) Perfectly on-brand (you control the CSS), (3) No DALL-E or Stability AI API costs, (4) Consistent visual quality.

For Phase 2, you can add AI image generation for custom hero images using FLUX or Google Imagen 3 — but this adds ~₹0.05–₹0.15/image. Include this in the Growth+ plans only to control cost.

What's the realistic first month revenue expectation?

Honest answer: ₹15,000–₹50,000 MRR in Month 1 with aggressive launch execution.

Based on your existing network response (colleagues asking for pricing + amazed at the product), you have a warm base. Conservative estimate: 30 Starter subscribers (₹499) + 10 Growth subscribers (₹999) = ₹14,970 + ₹9,990 = ~₹25,000 MRR at launch. This is above break-even from Day 1.

Ambitious but achievable: 50 Early Bird Starter (₹299) + 20 Early Bird Growth (₹649) + 5 Agency (₹3,999) = ₹14,950 + ₹12,980 + ₹19,995 = ~₹48,000 MRR. This requires a solid pre-launch campaign.

DO NOT count on 100+ subscribers in Month 1 without paid advertising. Organic launch caps at ~50 subscribers from personal network. First paid campaign is needed for the next wave.

My suggestion: Should I build a web dashboard too, or keep it pure WhatsApp?

Recommendation: Pure WhatsApp for first 6 months. Web dashboard in Month 7+.

Your positioning IS the WhatsApp-native experience. Adding a web dashboard dilutes the core proposition and adds significant development time. Users who want "no app, no website" are your exact market — don't break that promise.

A simple web page for: (1) Marketing landing page, (2) Razorpay payment checkout redirect, (3) Privacy policy and Terms — is fine and necessary. But not a full SaaS dashboard.

After 300+ subscribers, add a lightweight dashboard (NocoDB over your PostgreSQL is already in your stack) showing: usage history, content archive, brand kit management. This becomes a retention feature, not a core product.

What is the #1 thing that will kill this product if I get it wrong?

Content quality consistency. This is your make-or-break variable.

In the Indian market, pricing sensitivity is high but so is word-of-mouth. If users get great content — 60-70% of them will tell 2-3 others organically. If content disappoints — those same users will post publicly about it. You have zero margin for error on quality in the first 90 days.

Before launch: Test 50+ different topics across 5+ industries (real estate, coaching, textile, pharma, fashion) and ensure quality is consistently 80%+ comparable to a mid-level human copywriter. Build a quality rubric and test against it.

After launch: Build a "thumbs up/thumbs down" feedback loop in WhatsApp. After each content pack, send: "Rate this pack — 👍 or 👎". This gives you real-time quality data and shows users you care.
