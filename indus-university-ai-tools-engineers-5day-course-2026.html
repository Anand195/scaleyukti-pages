Design Philosophy

How This Course Is Built

Every design decision is driven by one goal: students should leave Day 5 having BUILT something with AI — not just learned about it.

⚡
Try It First, Explain Later

Every day opens with a 5-minute "Try It Now" challenge. Students use AI before they understand it — curiosity drives learning better than theory.

🌿
Branch-Specific Every Day

Every day has use cases for every branch. Civil engineers see civil examples. EC students see circuit code. No student ever feels the content is "for someone else."

📈
Pre-Work = Smooth Days

Each day ends with a 5-minute "Tonight's Setup" checklist. Students arrive ready to build — not waste 45 minutes on account creation and downloads.

🔨
Real Output Every Day

Every session ends with a shareable output — a report, resume, code script, AI image, video, or working agent. Day 5: a live demo in front of everyone.

⚠️

Pre-Course Setup (Send This 2 Days Before Day 1)

Ask students to create free accounts BEFORE arriving: ChatGPT (chat.openai.com) | Google Gemini (gemini.google.com) | Claude (claude.ai) | NotebookLM (notebooklm.google.com) | GitHub (github.com — use Indus University email) | Google Colab (colab.research.google.com). This saves 30–45 minutes of setup time on Day 1.

Session 3: Prompt Engineering Workshop — The Core Skill
🛠️ Hands-On

GitHub Gist QR Code (share at session start): Students scan QR → open Gist → ready-made prompts + incomplete prompts they must complete. No copying — they apply understanding.

The 5 Elements of a Great Prompt:

  • Role — "Act as a senior mechanical engineer..."
  • Context — "I am a 3rd year student working on a final year project about..."
  • Task — "Write / Generate / Explain / Analyse..."
  • Format — "...in bullet points / table / under 200 words..."
  • Constraint — "...without jargon / suitable for a non-engineer reader..."

Hands-On Challenge (40 min): Each student completes 3 branch-specific incomplete prompts from the Gist. They compare weak prompt vs strong prompt output. Competition: who gets the most useful response in 3 tries?

Branch Prompt Topics (from Gist):

💻 CSE / CE / IT / ICT

Generate a software requirements spec for a student attendance app. (Incomplete version in Gist — students fill in the missing role + constraints.)

⚡ EC / Electrical

⚙️ Mechanical

Select the best material for a bicycle frame that must be lightweight and corrosion-resistant. (Missing: context + constraints.)

🏗️ Civil

Session 3: n8n Automation — Build a Real AI Agent (3 Hours — Main Event)
🛠️ Hands-On

This is the most valuable session of the 5-day course. Students leave with a working AI agent on their phone.

Part A — n8n Orientation (20 min):

  • What is n8n? Visual workflow builder — connect tools without code.
  • Anatomy of a workflow: Trigger → Node → Node → Output
  • Run first workflow: manual trigger → set a variable → send a message. Everyone does this in 5 minutes.

Part B — Build a Job Hunt Agent OR Research Agent (90 min):

Students choose the agent most relevant to their current need. Both agents use the same n8n workflow structure.

Option A — Job Hunt Agent

Trigger (scheduled daily) → search job boards for specific role + location → AI filters by branch-relevant keywords → formats results as digest → sends to Telegram bot. Runs automatically every morning.

Option B — Research Agent

Trigger (user sends topic to Telegram bot) → agent searches web for latest papers/news on that topic → AI summarises in 5 bullet points → sends summary back to Telegram. Instant research assistant.

💻 CSE / CE / IT / ICT

Research agent: sends daily GitHub trending repos summary. Job hunt: filters software developer + data analyst roles, India + remote.

🏗️ Civil

📱 Telegram Bot Connection — Everyone Builds This Live

Every student connects their n8n agent to a personal Telegram bot. Result: they can interact with their AI agent from their phone, anywhere.

  1. Open Telegram → search @BotFather → send /newbot → name it (e.g. "MyResearchBot") → copy the API token
  2. In n8n: add a Telegram Trigger node → paste your bot API token → set trigger on "message"
  3. Add an AI node (connect to Gemini or ChatGPT API) → pass the user's message as input
  4. Add a Telegram Send Message node → send AI response back to user's chat ID
  5. Test: send a message to your bot → workflow fires → AI response comes back in under 5 seconds

End state: Every student has a working AI agent they can access from their Telegram app. They built it themselves in under 90 minutes.

📚 Concept

Run this AFTER the n8n hands-on — not before. Students already understand it through experience. Now give it the vocabulary.

  • What is an AI agent? — AI that takes actions, makes decisions, and completes goals autonomously. Not just answering questions — doing things.
  • "What you just built in n8n IS an agentic workflow." — connect the concept to what they built. Trigger → reason → act → output. That's an agent.
  • Where it's going: Multi-agent systems — agents that spawn sub-agents for sub-tasks. Your job-hunt agent could have a "resume tailoring sub-agent" and an "email drafting sub-agent."
  • Industry stat: 40% of enterprise applications will embed AI agents by 2026 (IDC). This is the #1 skill the market is hiring for right now.
  • Their role as engineers: You don't need to build AI models — you need to know how to deploy, orchestrate, and direct them. That's what you did today.

AI Ethics + Governance — Structured Discussion
📚 Discussion

Not a lecture. Run as an interactive discussion with real scenarios. Each scenario: "What do you think? Ethical or not? Why?"

  • Academic Integrity: "Should AI write your final year project report? Where's the line?" (Indian university context)
  • Data Privacy: "A company trains AI on employee emails without consent — acceptable?" (reference: Indian IT Act)
  • Deepfakes: Show 30-second real example → "Who is responsible for what AI creates?"
  • Bias in AI: "AI resume screener rejects female candidates more often. Who is responsible — developer, company, or AI?"
  • Job Displacement: "Will AI take engineering jobs? What's the new role of an engineer in 2030?"
Key Takeaway for Students

AI is a tool. Like any tool — a hammer, a calculator, a car — it amplifies what humans intend to do with it. The ethics sit with the human using the tool, not the tool itself. As engineers, your job is to build responsibly, deploy thoughtfully, and ask of every system you create: "Who does this help? Who could this harm?"

AI Ethics

Ethics Discussion Scenarios

Run as an interactive discussion — not a lecture. Each scenario takes 5–8 minutes. No right or wrong answer: the goal is critical thinking.

Academic Integrity

When does using AI for your thesis cross the line from "tool assistance" to "academic dishonesty"?

Scenario"A student uses Claude to write 80% of their final year project report. They understand the content and can explain it. Is this acceptable?"
Data Privacy

What happens when AI systems are trained on data people didn't know would be used?

Scenario"A company in Ahmedabad trains an AI model on 5 years of employee performance reviews — without telling employees. Legal under current Indian IT law?"
Deepfakes

AI can now generate realistic video of anyone saying anything. Where does responsibility lie?

Scenario"Someone uses Veo 3 to create a fake video of a professor saying something inappropriate and shares it. Who is legally responsible — the tool, the creator, or the platform?"
Algorithmic Bias

AI systems inherit the biases of their training data. What does that mean for engineering decisions?

Scenario"An AI resume screening tool rejects female candidates at a 30% higher rate than male candidates for engineering roles. Who is responsible — the developer, the company using it, or the AI?"
Job Displacement

AI is automating tasks that engineers used to do manually. Is this a threat or an opportunity?

Scenario"A civil engineering firm replaces 5 junior engineers with 1 AI system for structural calculations. The AI is more accurate and 100x faster. Is this progress or a problem?"
AI in Critical Systems

What level of AI decision-making is acceptable in systems where errors have life-or-death consequences?

Scenario"An AI system monitors a dam's structural sensors and can autonomously trigger evacuations. It triggers a false alarm, causing a stampede. Who is accountable?"
Key Takeaway for Students

AI is a tool. Like any tool — a hammer, a calculator, a car — it amplifies what humans intend to do with it. The ethics sit with the human using the tool, not the tool itself. As engineers, your job is to build responsibly, deploy thoughtfully, and ask of every system you create: "Who does this help? Who could this harm?"
