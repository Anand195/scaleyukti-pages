Design Philosophy

How This Course Is Built

Every design decision is driven by one goal: students should leave Day 5 having BUILT something with AI — not just learned about it.

⚡
Try It First, Explain Later

Every day opens with a 5-minute "Try It Now" challenge. Students use AI before they understand it — curiosity drives learning better than theory.

🌿
Branch-Specific Examples

Civil engineers see civil examples. ECE students see circuit code. Mechanical students see material selection. No generic examples that feel irrelevant.

📈
Progressive Complexity

Day 1: Everyone types a prompt. Day 5: Everyone builds a working prototype. Complexity builds gradually — no student left behind, no student bored.

🔨
Build, Not Watch

Instructor demos are max 15 minutes. Then students do it themselves. Every session ends with a shareable output — image, report, code, or automation.

⚠️

Pre-Course Setup (Send This 2 Days Before Day 1)

Ask students to create free accounts BEFORE arriving: ChatGPT (chat.openai.com) | Google Gemini (gemini.google.com) | Claude (claude.ai) | Canva (canva.com) | Gamma (gamma.app) | Make.com (make.com) | Kling AI (klingai.com). This saves 30–45 minutes of setup time on Day 1.

5-Day Curriculum

Session-by-Session Breakdown

Each day has a clear theme, warm-up challenge, structured sessions, and a tangible end-of-day outcome students take home.

1
AI Foundations + Prompt Engineering Mastery
Theme: "What AI can do TODAY — blow their minds first, explain after"
🎯Theory: 25% | Practical: 75%
⏱️Total: 6–7 hours
🛠️ChatGPT · Gemini · Claude · Copilot
⚡ Day 1 Warm-Up (First 5 minutes — before any introduction)

Everyone opens ChatGPT or Gemini. Task: "Ask AI to explain [your engineering branch] to a 10-year-old in 3 sentences." Read out 3 responses. This immediately creates curiosity and breaks the ice — even the most skeptical student is now engaged.

9:15 – 10:00 AM
Session 1: What Is AI, ML, and Generative AI?
📚 Theory🎬 Demo
  • AI vs Traditional Programming — show one visual: "Rules-based: IF brake → STOP" vs "ML: learns from 1 million driving examples"
  • ML vs GenAI — ML predicts patterns, GenAI creates new content. 2-minute explanation max, then demo.
  • Live Demo: Ask ChatGPT to solve an actual question from their last semester exam — show speed and capability
  • Live Demo: Show a hallucination — AI confidently gives wrong answer. Explain why critical thinking still matters.
  • Discussion: "In your branch, which task takes you the most time?" — collect answers, show AI can help with most of them
10:00 – 11:00 AM
Session 2: AI Impact by Branch — Specific to Their World
🎬 Demo

10 minutes per branch cluster. Show real use cases, not theory.

  • CSE / IT / CE / ICT: AI writes code → tests it → deploys it. GitHub Copilot live demo: generate a Python function in 10 seconds
  • EC / Electrical: AI generates Arduino sensor code, explains circuit faults, writes MATLAB signal processing scripts
  • Mechanical: AI does material selection analysis, generates CAD descriptions, creates preventive maintenance schedules
  • Civil: AI calculates structural loads, generates site risk reports, estimates material quantities from building specs
11:00 AM – 12:30 PM
Session 3: Prompt Engineering Workshop — The Core Skill
🛠️ Hands-On

The 5 Elements of a Great Prompt:

  • Role — "Act as a senior mechanical engineer..."
  • Context — "I am a 3rd year student working on..."
  • Task — "Write / Generate / Explain / Analyse..."
  • Format — "...in bullet points / table / 200 words..."
  • Constraint — "...without using jargon / use only free tools..."

Hands-On Challenge (40 min): Each student gets a branch-specific prompt task. They run 3 progressively better versions of the same prompt and see how output quality changes. Competition: "Who gets the most useful output in 3 prompts?"

1:30 – 3:30 PM
Session 4: AI Tools Tour + Hands-On Challenge
🛠️ Hands-On
  • Quick Comparison (10 min): ChatGPT vs Gemini vs Claude vs Copilot — same question, different outputs. Show strengths of each.
  • Branch Challenge (80 min): Each student picks ONE tool and solves ONE real branch-specific problem. Must be something they actually encounter in their studies.
  • Share Out (20 min): 5 students present their outputs on the projector — class votes on best prompt + best output
✅ Day 1 Student Outcome

Every student has successfully prompted an AI to solve a branch-relevant problem. They understand the difference between a weak prompt and a strong one. They have accounts on 3+ AI tools ready for tomorrow.

⏱️Total: 6–7 hours

9:15 – 10:00 AM
Session 1: AI-Assisted Coding Fundamentals
🎬 Demo🛠️ Hands-On
  • The New Coding Workflow: You describe what you want → AI writes the code → You test it → AI debugs it → Done.
  • Demo: GitHub Copilot — autocomplete code as you type. Show how it completes functions automatically.
  • Demo: ChatGPT vs Claude for code — same Python task, compare output quality
  • Google Colab intro — free Python runtime in browser, zero setup required. Every student can use it on their laptop.
  • Challenge (20 min): Run their warm-up code, see the output, modify it with AI assistance
10:00 – 12:00 PM
Session 2: Branch-Specific Coding Tracks (PARALLEL SESSIONS)
🛠️ Hands-On

Students split into branch groups. Instructor circulates between groups. Each group has a different track matching their engineering background.

💻 CSE / IT / CE / ICT

Build a Python web scraper → AI generates the code → students run + modify it. Then: build a simple chatbot using ChatGPT API (free tier). Advanced: a basic automation script using Python.

⚡ EC / Electrical

Generate Arduino/ESP32 code for a temperature sensor + LED alert using AI. Then: Python code in Colab to read, plot, and analyse simulated sensor data (voltage, current). No hardware needed — simulate data.

⚙️ Mechanical Engineering

AI generates Python code for beam bending moment calculation + plots graph in Colab. Then: MATLAB code generation for a simple dynamic system simulation. Students who don't know MATLAB can still run AI-generated code.

🏗️ Civil Engineering

Python code for structural load calculation + safe design check. AI generates it in Colab. Then: Python data analysis of survey data (input: Excel-style data → output: charts + summary). No prior Python knowledge required.

12:00 – 12:30 PM
Session 3: AI Debugging Workshop
🛠️ Hands-On

Instructor provides 5 pre-written buggy code files (branch-specific). Students use AI to: (1) Identify all bugs, (2) Fix them, (3) Explain why each bug exists. Speed challenge: who fixes all 5 fastest? Key insight: AI is a better debugger than most humans.

1:30 – 3:30 PM
Session 4: Branch-Specific Practical Use Case Build
🛠️ Hands-On

Each branch group builds ONE small working tool relevant to their engineering problems. Examples given to each group — they can also propose their own. Must be running and producing output by end of session. Present to class in last 30 minutes.

✅ Day 3 Student Outcome

Every student — even non-coders from Civil and Mechanical — has written and run code using AI assistance. They understand that coding knowledge + AI = superpowers. They have a working branch-specific mini script to show.

5
Mini Project Hackathon + AI Ethics + Certification
Theme: "You ARE an AI engineer now — prove it with something that works"
🎯Theory: 10% | Practical: 90%
⏱️Total: 7 hours (incl. ceremony)
🛠️Any 2+ tools from Days 1–4
⚡ Day 5 Warm-Up (5 minutes)

Quick round: Each student names ONE AI tool they discovered this week that surprised them. Then: Groups officially confirmed, project ideas locked. Hackathon clock starts.

9:15 AM – 12:15 PM
Mini Project Hackathon — 3 Hours STRICT
🛠️ Hands-On

Rules:

  • Teams: 3–4 students (mixed branches encouraged)
  • Must use minimum 2 AI tools from the course
  • Deliverable: Working live demo (not slides, not theory)
  • Demo must be max 3 minutes — show input, show AI doing work, show output
  • Instructor circulates every 30 minutes to help stuck teams

Judging Criteria: Does it work? → Is AI doing the actual work? → Does it solve a real problem? → Is the demo clear?

12:15 – 1:15 PM
Lunch + Final Demo Prep

Teams polish their demo during lunch. No new features — just make sure it works reliably.

1:15 – 2:30 PM
Live Demo Showcase — No Slides Allowed
🛠️ Hands-On
  • Each team: 3-minute live demo on screen — input → process → output, live in front of everyone
  • 1 minute for Q&A from instructor / audience
  • NO PowerPoint, NO "we planned to..." — only what actually works is shown
  • Top 3 projects get special recognition mention on certificate
2:30 – 3:15 PM
AI Ethics + Governance — Structured Discussion
📚 Discussion

NOT a lecture. Run as an interactive discussion with real scenarios. Each scenario: "What do you think? Ethical or not? Why?"

  • Academic Integrity: "Should AI write your final year project report? Where's the line?" (Indian university context)
  • Data Privacy: "A company trains AI on employee emails without consent — acceptable?" (reference: Indian IT Act)
  • Deepfakes in India: Show 30-second real example → "Who is responsible for what AI creates?"
  • Bias in AI: "AI resume screener rejects female candidates more often. Who is responsible — developer, company, or AI?"
  • Job Displacement: "Will AI take engineering jobs? What's the new role of an engineer?"
3:15 – 4:30 PM
Certificate Distribution + Closing
  • Course feedback form (use Google Forms + AI to auto-analyse results)
  • Recognition of top 3 projects
  • Certificate distribution — "AI Tools for Engineers" certificate signed by instructor + institution
  • Group photo + LinkedIn tips: how to add certificate + skills to profile
✅ Day 5 Student Outcome

Every student has built a working AI prototype with their team. They have a certificate, a project to put on their resume, and know exactly how to add AI skills to their LinkedIn profile. They leave as AI practitioners — not just AI-aware students.

Day 5 — Ethics Discussion

AI Ethics + Governance: 5 Discussion Scenarios

NOT a lecture. Run as a structured group discussion. Each scenario: share the case → ask "What do you think? Ethical or not? Who is responsible?" → discuss for 5 minutes → instructor summarises.

1. Academic Integrity — Where's the Line?

A student uses ChatGPT to write their entire final year project report, edits 10%, and submits it as their own work. Is this plagiarism? What about using AI to improve their own written draft?

Discussion Anchor"If a student used a calculator in an exam, is that cheating? How is AI different? Where exactly should the line be?"
2. Data Privacy — Your Data Training Their AI

When you use free AI tools, your prompts and data may be used to train their models. You just pasted your college's confidential project proposal into ChatGPT. Is this acceptable?

Discussion Anchor"What personal or professional information should you NEVER put into a free AI tool? How does India's DPDP Act 2023 protect you?"
3. Deepfakes — Real Harm, Digital Origin

An AI-generated video of a politician saying something they never said goes viral on WhatsApp in India before an election. Who is legally and morally responsible — the person who made it, the platform that hosted it, or the AI company?

Discussion Anchor"As engineers who now know how to use AI image and video generation — what is YOUR responsibility when you have this power?"
4. AI Bias — When Algorithms Discriminate

An AI hiring tool used by a large Indian IT company rejects women candidates at 2x the rate of men. The company says "the algorithm did it." The algorithm was trained on 10 years of their own past hiring data.

Discussion Anchor"Who is responsible — the algorithm, the company, the developers, or the people who generated the biased training data?"
5. AI & Jobs — Threat or Opportunity?

A manufacturing plant in Gujarat replaces 50 assembly workers with AI-controlled robotic arms. The owner says it reduces costs and errors. The workers lose their livelihood.

Discussion Anchor"You are now AI-capable engineers. Are you part of the automation wave displacing others? What is your ethical responsibility as you build AI systems?"
Key Takeaway for Students

AI is a tool. Like any tool — a hammer, a calculator, a car — it amplifies what humans intend to do with it. The ethics sit with the human using the tool, not the tool itself. As engineers, your job is to build responsibly, deploy thoughtfully, and question every system you create: "Who does this help? Who could this harm?"
